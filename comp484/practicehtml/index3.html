embedded styling

styles that are placed in a style element in the head section use selectors to apply style elements throughout the entire document. ie <style type = "text/css">. which is inside the html file, init in the head section. whichever element that has more specificity, the style will override the style given to it.

    css selector
  • .class-name { styles }

    • the class attribute is mainly used to styling. It lets you organize your css file. class selectors in the css file should be

  • element { styles }

    • selects all element

  • #idname { styles }

    • is id selector in the css.

  • * { styles }

    • is a way to select all the elements for css.

  • element1, element2 { styles }

    • selects all element1 and element2

  • element1 element2 { styles }

    • selects all element2 inside element1, nested elements

  • background-color: inherit; // background-color: initial;

    • inherit is a way to get the styles from that elements' parent. initial is a way to get the style for that element to not inherit from its parent, But setting initial does not always override the inheritance.

  • element1>element2 { styles }

    • selects all element 2 where the parent is element1, direct child

  • element1+element2 { styles }

    • selects all element2 that are placed immediately after element1, this is more strict than ~

  • element1~element2 { styles }

    • selects every element2 that are preceded by a element1, general siblings of element1

  • [attribute] ie. input[type="text"]

    • selects all elements with a target attribute, the example combines both element selector and attribute selector

notes

conflicting styles
styles may be defined by a user (person viewing your webpage), an author (whoever is writing the doc) or a user agent (browser default style). styles defined by the user take precedence over styles defined by the user agent. styles defined by authors take precedence over styles defined by the user. (User important) > Author > user > user-agent.
some conflicting styles are
    text decorations
  • line-through
  • underlined
  • overline
properties defined for child and descendant elements have a higher specificity than properties defined for parent and ancestor elements. specificity values go from Most [style attribute, ID, class/ pseudo-class attribute, elements] Least. In the presence of 2 conflicting rules with the same specificity, order will choose the precedence.
pseudoclasses give you access to content that's not declared in the document. It can be used to style an element when a user mouse moves over it which is hover, or when the element gets focus. for example -> a:link{ color: #FFF; } or a:hover{ color: #0F0; }
pseudoclass selector can be combined with a css class ex. -> a.class-name:hover{ color: #F00; }
Always use relative length measurements ie. px (pixels), em (usually the height of M), ex (usually the height of x), and percentages (of the font's default size).
linking external style sheets
external style sheets are separate documents that contain only CSS rules. ie. in a file called styles.css This helps create a uniform look for a website. To link a css file to html doc, add under head section of the html file.
<link rel = "stylesheet" type = "text/css" href = "./file-directory/name-of-css-file.css" >
positioning elements:
Absolute positioning provides greater control over where on a page elements reside.
  • static: html elements are positioned static by default. arent affected by the top, bottom, left, right properties.
  • relative: the element is positioned relative to its normal position
  • fixed: the element is positioned relative to the viewport whih means it always stays in the same place even if the page is scolled.
  • Absolute: is positioned relative to the nearest positioned ancestor (instead of positioned relative to the viewport, like fixed)
    • specifying an element's position as absolute removes it from the normal flow.
  • sticky
  • z-index is to tell the browser what to send to the background (lower index) and foreground (higher index)
elements are then positioned using the top, bottom, left, and right properties. However, these properties will not work unless the position property is set first. THey also work differently depending on the position value.

notes

Relative position does not remove the element from the normal flow of elements. Superscript and subscript example.
.super { position: relative; top: -10px; }
.sub { position: relative; top: -10px; }
.right { position: relative; top: 10px; }
.left { position: relative; top: -10px; }
inline and block level elements
inline doesn't change the flow of the document, ex of inline elements are img, a, em, span(which is a grouping element: doesnt apply any formatting to its contents. creates a container for CSS)
block-level elements are displayed on their own line ie. p, all headings (h1-h6), div(a grouping element).
CSS can control the backgrounds of block level elements by adding colors and images with flexible position and attachment and repitition.
background image's syntax is
background-image: url(logo.png); < -- gets image
background-repeat: no-repeat; < no repetition of image
background-position: bottom right; < position of image
background-attachment: fixed; < makes it so the image stays in place
which will be in the body{} of css

box model and text flow
block level html5 elements have a virtual box drawn around them based on the css box model.
